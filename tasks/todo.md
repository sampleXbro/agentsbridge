## 1. Scope and canonical surface

The **only** canonical inputs under `.agentsmesh/` are these 8 feature families (`src/core/canonical-types.ts:106`):

| Family | Canonical source | Notes |
|--------|------------------|-------|
| rules | `rules/*.md` (`CanonicalRule`) | Has `root`, `targets[]`, `globs[]`, `codexEmit` |
| commands | `commands/*.md` (`CanonicalCommand`) | Has `outputStyle` flag |
| agents | `agents/*.md` (`CanonicalAgent`) | Has `outputStyle` flag |
| skills | `skills/<name>/SKILL.md` (`CanonicalSkill`) | Supporting files allowed |
| mcp | `mcp.json` (`McpConfig`) | |
| permissions | `permissions.yaml` (`Permissions`) | allow / deny / ask |
| hooks | `hooks.yaml` (`Hooks`) | |
| ignore | `ignore` (`IgnorePatterns`) | |

**Rule — gap in scope only if:**
1. A canonical input above maps to a target output the doc describes, AND
2. The generator fails to emit it, emits it wrong, or suppresses it without doc authority.

**Target-native features without canonical source are out of scope** even if the `docs/agent-structures/*` file mentions them (they belong on a product roadmap, not in this audit). See §5.
