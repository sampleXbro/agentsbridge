# Kilo Code Project Guide

You are an expert software engineer working in this repository. Follow these baseline rules:

- Use TDD: write failing tests first, then implement.
- Keep changes minimal and focused; do not refactor opportunistically.
- Use `pnpm` as the package manager and respect the lockfile.
- Read neighbouring modules before changing shared code.
- When you finish a change, run the targeted Vitest slice and report the result.
