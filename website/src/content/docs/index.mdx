## AI coding config should not drift between tools

Every AI coding assistant ships a different configuration surface. Claude Code reads `CLAUDE.md` and `.claude/settings.json`. Cursor uses `.cursor/rules/*.mdc`. GitHub Copilot reads `.github/copilot-instructions.md` and prompts. Gemini CLI, Windsurf, Continue, Cline, Kiro, Codex CLI, Junie, Roo Code, Antigravity, Amp, Zed, and Warp each add their own files for rules, commands, agents, skills, MCP servers, hooks, ignore patterns, or permissions.

AgentsMesh turns that spread into one editable source of truth. Put the real project instructions in `.agentsmesh/`, then generate native files for every enabled tool. Import goes the other direction, so existing tool configs can be adopted without starting from scratch.

## One canonical directory

```bash
.agentsmesh/
  rules/_root.md
  rules/*.md
  commands/*.md
  agents/*.md
  skills/*/SKILL.md
  mcp.json
  hooks.yaml
  permissions.yaml
  ignore
```

```bash
agentsmesh generate
# writes CLAUDE.md, AGENTS.md, .cursor/rules/, .github/copilot-instructions.md,
# .gemini/settings.json, .windsurf/rules/, .codex/config.toml, .kiro/steering/,
# .cline/, .junie/, .roo/, .agents/, and more from the same source.
```

<Aside type="tip">
  When a target lacks native support for a feature, AgentsMesh can project it into a compatible format and embed metadata so `agentsmesh import` can restore the canonical file later.
</Aside>

## Quick start

### New project

```bash
npm install -D agentsmesh
npx agentsmesh init
# edit .agentsmesh/rules/_root.md
npx agentsmesh generate
```

### Existing project

```bash
npx agentsmesh import --from cursor
npx agentsmesh generate
```

### Personal global config

```bash
npx agentsmesh init --global
npx agentsmesh import --global --from claude-code
npx agentsmesh generate --global
```

The CLI is available as both `agentsmesh` and `amsh`.
