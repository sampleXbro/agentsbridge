## The Problem

Every AI coding tool has its own config format. `.claude/rules/`, `.cursor/rules/`, `.github/copilot-instructions.md`, `.gemini/`, `.cline/`... Change a rule in one place and forget the others. Now your agents behave differently depending on who's editing.

Scale that across a team where some use Cursor, some use Claude Code, and some switch between them ÔÇö and config drift becomes a real problem.

## The Solution

```bash
# One directory, all your AI tool config
.agentsmesh/
  rules/_root.md     # ÔåÉ edit here
  commands/*.md
  agents/*.md
  skills/*/SKILL.md
  mcp.json
  hooks.yaml
  permissions.yaml
  ignore
```

```bash
# Generate correct output for every tool
agentsmesh generate

# Claude Code, Cursor, Copilot, Gemini CLI, Cline,
# Codex CLI, Windsurf, Continue, and Junie ÔÇö all in sync.
```

## Why AgentsMesh

<CardGrid>
  <Card title="One Source of Truth" icon="pencil">
    Edit `.agentsmesh/`, generate everywhere. No more copy-pasting rules between tool directories. No more drift.
  </Card>
  <Card title="Bidirectional Sync" icon="sync">
    Import existing tool configs into canonical form, then generate back out. Lossless round-trip ÔÇö no manual reformatting.
  </Card>
  <Card title="CI Drift Detection" icon="warning">
    `agentsmesh check` exits with code 1 if generated files have drifted from the lock. Catch drift in PRs before it ships.
  </Card>
  <Card title="Lossless Projection" icon="puzzle">
    When a tool lacks native support for a feature, AgentsMesh embeds it with enough metadata to round-trip on re-import. No data loss.
  </Card>
  <Card title="Team-Safe Collaboration" icon="group">
    Lock files track generated state. `agentsmesh merge` resolves conflicts after `git merge`. Collaboration settings prevent accidental overrides.
  </Card>
  <Card title="Real Operational Tooling" icon="setting">
    `diff`, `lint`, `watch`, `check`, `merge`, and `matrix` are first-class commands ÔÇö not afterthoughts.
  </Card>
</CardGrid>

## Quick Start

```bash
# Install
npm install -D agentsmesh

# Scaffold the canonical directory
npx agentsmesh init

# Edit your root rule
# vi .agentsmesh/rules/_root.md

# Generate for all configured targets
npx agentsmesh generate
```

**Already have tool configs?** Import them first:

```bash
npx agentsmesh import --from cursor
npx agentsmesh import --from claude-code
npx agentsmesh generate
```
