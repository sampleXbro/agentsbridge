## The problem — every AI coding tool has its own config

Claude Code reads `CLAUDE.md`. Cursor wants `.cursor/rules/*.mdc`. GitHub Copilot expects `.github/copilot-instructions.md`. Gemini CLI looks at `GEMINI.md` and `.gemini/settings.json`. Windsurf needs `.windsurf/rules/`. Codex CLI uses `AGENTS.md` and `.codex/config.toml`. Every tool adds its own layout for rules, commands, agents, skills, MCP servers, hooks, ignore files, and permissions.

Change a rule in one place, forget the others, and behavior drifts between editors. Scale that across a team where one developer uses Cursor, another Claude Code, a reviewer Copilot — and keeping instructions aligned becomes a maintenance problem.

## The solution — one canonical directory, every tool

```bash
.agentsmesh/
  rules/_root.md       # ← edit once
  rules/*.md
  commands/*.md
  agents/*.md
  skills/*/SKILL.md
  mcp.json
  hooks.yaml
  permissions.yaml
  ignore
```

```bash
agentsmesh generate    # produces CLAUDE.md, .cursor/rules/, .github/copilot-instructions.md,
                       # .gemini/settings.json, .windsurf/rules/, AGENTS.md, .codex/config.toml,
                       # .kiro/steering/, .cline/, .junie/, .roo/, .agents/, and more
```

AgentsMesh gives teams one editing surface, one lock-checked output set, and one import/generate workflow that keeps every environment aligned — with **bidirectional sync** so you can adopt it on existing projects without losing work.

## Quick start

### New project

```bash
npm install -D agentsmesh
npx agentsmesh init
# edit .agentsmesh/rules/_root.md
npx agentsmesh generate
```

### Existing project — adopt with one import

```bash
npx agentsmesh import --from cursor       # or claude-code, copilot, codex-cli, gemini-cli, ...
npx agentsmesh generate
```

### Personal global config

```bash
npx agentsmesh init --global
npx agentsmesh import --global --from claude-code
npx agentsmesh generate --global          # writes to ~/.claude/, ~/.cursor/, etc.
```

<Aside type="tip">
  The CLI is available as both `agentsmesh` and the shorter alias `amsh`.
</Aside>

## Three ways to use AgentsMesh

### 1. Team config (project mode)

```bash
agentsmesh init
agentsmesh generate                       # outputs to .claude/, .cursor/, .github/, etc.
agentsmesh check                          # CI gate — exit 1 on drift
```

One `.agentsmesh/` directory in your repo, committed with code. Every developer on the team gets the same rules, skills, agents, and MCP servers regardless of which tool they use. See the [multi-tool teams guide](./guides/multi-tool-teams/) for patterns.

### 2. Personal setup (global mode)

```bash
agentsmesh init --global
agentsmesh import --global --from claude-code
agentsmesh generate --global              # syncs to ~/.claude/, ~/.cursor/, ~/.codex/, ...
```

One `~/.agentsmesh/` directory for your personal tooling across every repo. Every built-in target and every plugin supports global mode. See the [supported tools matrix](./reference/supported-tools/#global-mode) for per-tool paths.

### 3. Extensibility (plugins)

```bash
agentsmesh plugin add agentsmesh-target-my-tool
agentsmesh plugin list                    # show registered plugins + load status
agentsmesh generate                       # plugins run alongside built-ins
```

Plugins are npm packages that export a `TargetDescriptor`. Full parity with built-in targets: project + global mode, feature conversions, scoped settings, per-feature lint hooks, and hook post-processing. See the [Building Plugins](./guides/building-plugins/) guide for a complete walkthrough.

## Broad tool coverage

AgentsMesh ships native support for every major AI coding tool — with native mappings where the tool allows, and lossless embedded projections where it doesn't:

**Claude Code** (`CLAUDE.md`, `.claude/agents/`, `.claude/skills/`, `.claude/commands/`, `.claude/settings.json`, `.claude/hooks.json`) · **Cursor** (`.cursor/rules/*.mdc`, `AGENTS.md`, `.cursor/mcp.json`) · **GitHub Copilot** (`.github/copilot-instructions.md`, `.github/agents/`, `.github/prompts/`) · **Gemini CLI** (`GEMINI.md`, `.gemini/settings.json`, `.gemini/commands/*.toml`) · **Windsurf** (`.windsurf/rules/`, `.windsurf/workflows/`, `.windsurf/mcp_config.json`) · **Continue** (`.continue/rules/`, `.continue/prompts/`, `.continue/config.yaml`) · **Cline** (`.clinerules/`, `.cline/skills/`, MCP, hooks) · **Kiro** (`.kiro/steering/`, `.kiro/skills/`, `.kiro/hooks/`) · **Codex CLI** (`AGENTS.md`, `.codex/config.toml`, `.codex/agents/*.toml`) · **Junie** (`AGENTS.md`, `.junie/agents/`, `.junie/commands/`, `.junie/skills/`) · **Roo Code** (`.roo/rules/`, `.roomodes`, `.roo/commands/`) · **Antigravity** (`.agents/rules/general.md`, `.agents/skills/`, `.agents/workflows/`)

See the **[full feature matrix](./reference/supported-tools/)** for native vs. embedded support per tool, or run [`agentsmesh matrix`](./cli/matrix/) locally to see what your config produces.

## Install community skills in seconds

```bash
agentsmesh install github:my-org/ai-skills@v1.0.0
agentsmesh install github:team/standards --path rules --as rules
agentsmesh install --sync                 # restore packs after a fresh clone
```

Installed packs live in `.agentsmesh/packs/`, are tracked in `.agentsmesh/installs.yaml`, and merge into canonical config on every `generate`. See the [install docs](./cli/install/) and [community packs guide](./guides/community-packs/).
