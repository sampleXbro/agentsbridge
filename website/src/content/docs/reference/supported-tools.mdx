AgentsMesh supports every major AI coding tool. This page documents what each tool supports natively, what is embedded/projected, and what is not supported.

## Legend

| Symbol | Meaning |
|--------|---------|
| **Native** | Feature maps directly to the tool's own config format. Full fidelity. |
| **Embedded** | AgentsMesh projects the feature into a compatible format with metadata for lossless round-trip on import. |
| **Partial** | Limited support — some fields or capabilities may not fully round-trip. |
| **—** | Not supported. Feature is skipped for this target. |

## Global mode

With `--global`, the canonical directory is `~/.agentsmesh/` and CLI commands resolve user-level paths from `$HOME`. **Every** built-in target implements global mode (`descriptor.globalSupport`). The **project** and **global** feature matrices below match `SUPPORT_MATRIX` and `SUPPORT_MATRIX_GLOBAL` in `src/core/matrix/data.ts` (normalized from each target’s capabilities vs `globalSupport.capabilities`). The [Global mode](#global-mode) paths table and each tool section document user-level paths and limitations.

| Target | Global generate writes | Global import reads |
|--------|------------------------|---------------------|
| Claude Code | `~/.claude/**` (including `CLAUDE.md`, `settings.json`, `hooks.json`, `skills/`, `agents/`, `commands/`, `output-styles/`), `~/.claude.json`, `~/.claudeignore`, mirrored skills under `~/.agents/skills/` | Same user-level paths (hooks prefer `~/.claude/hooks.json` when present) |

## Tool details

### Claude Code

**Output directory:** `.claude/`

Full native support for all 8 features. The reference implementation for AgentsMesh's canonical format.

| Feature | Format |
|---------|--------|
| Rules | `.claude/rules/*.md` |
| Commands | `.claude/commands/*.md` |
| Agents | `.claude/agents/*.md` |
| Skills | `.claude/skills/*/SKILL.md` |
| MCP Servers | `.claude/settings.json` → `mcpServers` |
| Hooks | `.claude/settings.json` → `hooks` |
| Ignore | `.claudeignore` |
| Permissions | `.claude/settings.json` → `permissions` |

Global mode is fully supported for Claude Code: canonical `~/.agentsmesh/` drives user-level outputs under `~/.claude/`, `~/.claude.json`, and `~/.claudeignore`. Hooks emit to `~/.claude/hooks.json` (project mode keeps hooks in `settings.json`). Skills emit to `~/.claude/skills/` and, when **`codex-cli` is not** among global targets, a compatibility mirror under `~/.agents/skills/` (if Codex is enabled globally it already owns that tree, so Claude does not duplicate there). Agents or commands with frontmatter `outputStyle: true` also emit `~/.claude/output-styles/agent-{name}.md` or `command-{name}.md`. Root rules render under `# Global Instructions` in `~/.claude/CLAUDE.md`. See [Global mode](#global-mode) for all targets.

### Cursor

**Output directory:** `.cursor/`

Full native support except permissions (partial).

| Feature | Format | Notes |
|---------|--------|-------|
| Rules | `.cursor/rules/*.mdc` | Converted to Cursor's `.mdc` format |
| Commands | `.cursor/commands/*.md` | |
| Agents | `.cursor/agents/*.md` | |
| Skills | `.cursor/skills/*/SKILL.md` | |
| MCP Servers | `.cursor/mcp.json` | |
| Hooks | `.cursor/hooks.json` | Cursor uses a standalone hooks file (not `settings.json`) |
| Ignore | `.cursorignore` | |
| Permissions | `.cursor/settings.json` | Partial field coverage |

**Global mode:** writes the same Cursor surfaces as project mode under `~/.cursor/` (per-rule `*.mdc`, `general.mdc`, aggregated `AGENTS.md`, MCP, hooks, ignore, skills, agents, commands). `agentsmesh import --global --from cursor` reads `~/.cursor/rules/*.mdc` first, then falls back to legacy `~/.agentsmesh-exports/cursor/user-rules.md` or `~/.cursor/AGENTS.md`, plus MCP, hooks, ignore, skills, agents, and commands. See `docs/agent-structures/cursor-global-level-generation-strategy.md` for the intended redundancy model.

| Feature | Format | Notes |
|---------|--------|-------|
| Rules | `AGENTS.md`, `.windsurf/rules/*.md` | Root in `AGENTS.md`; modular rules with frontmatter; global: `~/.codeium/windsurf/memories/global_rules.md` (aggregate) |
| Commands | `.windsurf/workflows/*.md` | Workflows; global: `~/.codeium/windsurf/global_workflows/` |
| Agents | `.windsurf/skills/` | Embedded as skills |

| Hooks | `.windsurf/hooks.json` | Full hook support; global: `~/.codeium/windsurf/hooks.json` |

### Kiro

**Output directory:** `.kiro/` plus root `AGENTS.md`

| Feature | Format | Notes |

| Permissions | — | No documented project permissions file |

### Roo Code

**Output directory:** `.roo/`

Roo Code supports native rules, commands, skills, MCP, and ignore. Per-mode rule directories (`.roo/rules-{mode}/`) are imported as regular canonical rules and are not generated back out on generate.

**Global mode:** `agentsmesh generate --global` writes to `~/.roo/rules/*.md`, `~/.roo/commands/*.md`, `~/.roo/skills/`, `~/mcp_settings.json` (Roo’s home-level MCP settings file), `~/.rooignore`, and `~/.roo/AGENTS.md` (compatibility aggregate). Skills are mirrored to `~/.agents/skills/` unless Codex CLI is also a global target.

| Feature | Format | Notes |
|---------|--------|-------|
| Rules | `.roo/rules/*.md` | Root rule emits as `.roo/rules/00-root.md` (alphabetically first); fallback import from `.roorules` |
| Commands | `.roo/commands/*.md` | `description` frontmatter preserved |
| Agents | — | Sub-agents are runtime-only (Boomerang Tasks); no project config surface |
| Skills | `.roo/skills/*/SKILL.md` | Native format with `name`, `description`, `modeSlugs` frontmatter |
| MCP Servers | `.roo/mcp.json` → `mcpServers` | Supports stdio and HTTP/SSE transports |
| Hooks | — | No hook system in Roo Code |
| Ignore | `.rooignore` | gitignore syntax |
| Permissions | — | Not supported |
